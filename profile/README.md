## What Is Tower for Mac

Tower for Mac is a Git client for macOS that lets you enjoy the full power of Git through an elegant,
comprehensive interface. Where the command line asks you to remember dozens of subcommands and flags,
tower mac turns everyday actions — staging, committing, branching, merging — into clear, visual steps.
Lots of operations can be performed simply via drag-and-drop, and when something goes wrong, mistakes
can be undone easily.

At the same time, it does not dumb Git down. As a git client mac professionals rely on, Tower exposes
advanced capabilities such as single-line staging, submodule support and file history, so power users
can increase their productivity rather than trade it away. This guide walks you through getting Tower
set up and working day to day.

## Getting Started

Follow these steps to go from download to your first commit with tower for mac.

1. **Download and install Tower.** Get download tower for mac, open the disk image, and drag the app
   into your Applications folder.
2. **Add your account.** Connect your hosting service — Tower offers direct GitHub, Bitbucket, GitLab,
   GitSwarm, Beanstalk, Visual Studio Team Services, Team Foundation Server and RhodeCode integration.
3. **Open a repository.** Clone an existing repo from your connected service, add a local folder that
   already uses Git, or create a brand-new repository.
4. **Explore the working copy.** Review changed files, view diffs, and get familiar with the branches
   list and commit history.
5. **Make your first commit.** Stage the changes you want — down to a single line if you like — write a
   message, and commit.

Within a few minutes, git gui mac work that used to mean memorizing commands becomes point-and-click.

## Configuring Tower

A little setup makes tower mac download fit the way you work.

**Connect multiple services**: Add each hosting provider you use so cloning and pushing stay one click
away. Bitbucket team accounts are supported alongside personal ones.

**Enable git-flow**: If your team follows the git-flow branching model, turn on git-flow support so
feature, release and hotfix branches are created and finished with guided actions.

**Set up submodules and LFS**: Tower includes submodule support and git-lfs support, so repositories
with nested modules or large binary files behave correctly from the start.

**Tune your display**: Support for full-screen and Retina displays keeps diffs and history crisp on any
Mac.

## Everyday Workflow

Once configured, most of your day in tower for mac revolves around a few smooth actions.

**Stage precisely**: Use single-line staging (per-line committing) to include exactly the changes you
mean, splitting unrelated edits into separate commits.

**Branch and merge by drag-and-drop**: Many powerful drag-and-drop features cover merge, rebase, and
push or pull to and from a remote, plus applying a stash or creating a new branch.

**Cherry-pick and refine**: Cherry-pick specific commits between branches, and discard chunks or lines
you do not want to keep.

**Investigate history**: Use file history to see how a file evolved, and the blame view to find exactly
when and why a line changed.

**Undo with confidence**: Because mistakes can be undone easily, you can experiment knowing a wrong move
is recoverable.

---

## Tips for Getting More from Tower

**Commit in small, logical pieces**: Single-line staging exists precisely so one messy working copy can
become several clean commits.

**Lean on drag-and-drop**: For merges, rebases and stash application, dragging is often faster and less
error-prone than typing.

**Keep git-svn projects in Tower too**: git-svn support means you can manage Subversion-backed projects
in the same git client mac interface.

**Use blame before you ask**: The blame view often answers "why is this line here?" faster than tracking
someone down.

## Frequently Asked Questions

**Do I need to know Git commands to use Tower?**
No. tower mac abstracts Git's complexity into a visual interface, and many actions are drag-and-drop —
though the full power of Git is still available when you want it.

**Which hosting services does it integrate with?**
Tower offers direct integration with GitHub, Bitbucket, GitLab, GitSwarm, Beanstalk, Visual Studio Team
Services, Team Foundation Server and RhodeCode, including Bitbucket team accounts.

**Can I commit only part of my changes?**
Yes. Single-line staging lets you commit per line, so you can split a working copy into clean, focused
commits.

**Does it support submodules and large files?**
It does. tower for mac includes both submodule support and git-lfs support out of the box.

**What if I make a mistake?**
Mistakes can be undone easily in Tower, so branching, merging and rebasing all stay low-risk.
